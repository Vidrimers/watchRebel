# Маршруты приложения watchRebel

## Обзор

Приложение использует React Router v6 с lazy loading для оптимизации загрузки страниц.

## Структура маршрутов

### Публичные маршруты
- `/login` - Страница авторизации через Telegram

### Защищенные маршруты (требуют авторизации)
- `/` - Главная страница (редирект на профиль пользователя)
- `/user/:userId` - Профиль пользователя
- `/search` - Поиск фильмов, сериалов и пользователей
- `/media/:mediaType/:mediaId` - Детальная страница контента (mediaType: movie|tv)
- `/lists` - Пользовательские списки
- `/watchlist` - Список желаемого
- `/notifications` - Уведомления
- `/settings` - Настройки пользователя

## Lazy Loading

Все страницы загружаются динамически с помощью `React.lazy()` и `Suspense`:

```javascript
const LoginPage = lazy(() => import('./pages/LoginPage'));
```

Это позволяет:
- Уменьшить размер начального бандла
- Ускорить первую загрузку приложения
- Загружать страницы только при необходимости

## Защита маршрутов

Защищенные маршруты используют компонент `ProtectedRoute`, который:
- Проверяет наличие авторизации
- Перенаправляет на `/login` если пользователь не авторизован
- Отображает страницу если пользователь авторизован

## Fallback при загрузке

При загрузке страниц отображается компонент `LoadingFallback` с текстом "Загрузка..."

## Обработка неизвестных маршрутов

Все неизвестные маршруты перенаправляются на главную страницу `/`
